### 1.3 操作逻辑流程

#### 创建任务流程
```
用户点击"新建任务" 
  → 打开 TaskEditDialog 组件
  → 填写表单数据
  → 调用 store.createTask()
  → 触发 IPC 'tasks:create'
  → main 进程 task-handlers.ts 处理
  → 执行 SQL INSERT
  → saveDatabase() 写入文件
  → 返回结果更新 store
  → UI 自动刷新
```

#### 更新任务流程
```
用户修改任务状态/进度
  → 调用 store.updateTaskStatus() / updateTaskProgress()
  → 触发 IPC 'tasks:update'
  → main 进程验证允许更新的字段
  → 执行 SQL UPDATE
  → saveDatabase() 写入文件
  → 返回结果更新本地 store
```

#### 删除项目流程
```
用户点击删除项目
  → 弹窗确认（含警告：任务会被级联删除）
  → 调用 store.deleteProject()
  → 触发 IPC 'projects:delete'
  → 执行 SQL DELETE（外键级联删除任务）
  → saveDatabase() 写入文件
  → 更新 store 和 currentProjectId
```

## 二、问题识别

### 2.1 数据存储问题

#### 🔴 严重问题

**1. 数据库性能瓶颈**
- **问题**：每次操作都调用 `saveDatabase()` 导出整个数据库并写入文件
- **影响**：数据量大时性能急剧下降，频繁 I/O 操作
- **风险**：应用崩溃时可能丢失数据（内存与文件不同步）

**2. 缺少事务支持**
- **问题**：没有使用 SQL 事务，批量操作可能部分成功
- **影响**：数据一致性问题（如创建任务失败但 ID 已生成）
- **风险**：数据损坏风险

**3. 缺少数据验证**
- **问题**：数据库层面缺少外键约束检查、触发器
- **影响**：可能插入无效数据（如引用不存在的项目）
- **风险**：数据完整性问题

#### 🟡 中等问题

**4. ID 生成策略不统一**
- **问题**：使用 `task_${Date.now()}_${random}` 格式，非标准 UUID
- **影响**：ID 可读性差，无法分布式生成
- **风险**：极端情况下可能重复（虽然概率极低）

**5. 时间戳处理混乱**
- **问题**：混用 `datetime('now')` 和 `new Date().toISOString()`
- **影响**：时区不一致，排序可能出错
- **风险**：跨时区用户数据混乱

**6. 缺少软删除**
- **问题**：直接物理删除，无法恢复
- **影响**：误删后无法找回
- **风险**：用户数据丢失投诉

### 2.2 操作逻辑问题

#### 🔴 严重问题

**1. 状态管理冗余**
- **问题**：Store 中同时维护 `tasks` 数组和多个操作方法
- **影响**：代码重复，更新逻辑分散
- **风险**：容易出现状态不一致

**2. 缺少乐观更新**
- **问题**：所有操作都等待数据库返回后才更新 UI
- **影响**：用户体验差，感觉卡顿
- **风险**：网络延迟时体验极差

**3. 错误处理不完善**
- **问题**：错误只存储在 store 的 `error` 字段，缺少全局处理
- **影响**：用户看不到错误提示
- **风险**：问题被忽略

#### 🟡 中等问题

**4. 表单验证缺失**
- **问题**：TaskEditDialog 缺少前端验证
- **影响**：可能提交无效数据
- **风险**：数据库错误

**5. 批量操作优化**
- **问题**：批量删除使用多个单独 DELETE 而非事务
- **影响**：性能差
- **风险**：部分删除失败

**6. 统计计算低效**
- **问题**：ProjectView 中在渲染时计算统计信息
- **影响**：每次渲染都重新计算
- **风险**：数据量大时卡顿

### 2.3 代码质量问题

#### 🟡 中等问题

**1. IPC 处理器过于臃肿**
- **问题**：task-handlers.ts 465 行，包含所有任务相关操作
- **影响**：难以维护和测试
- **建议**：拆分为独立 service 层

**2. 类型定义不完整**
- **问题**：preload 和 main 使用 `any` 类型
- **影响**：失去 TypeScript 类型安全
- **风险**：运行时错误

**3. 缺少单元测试**
- **问题**：没有测试文件
- **影响**：重构困难，回归风险高

// main/src/services/database.ts 优化
class DatabaseManager {
  private db: Database | null = null
  private saveTimeout: NodeJS.Timeout | null = null
  private readonly SAVE_DELAY = 1000 // 1 秒防抖

  // 延迟保存，避免频繁 I/O
  async saveDatabase() {
    if (this.saveTimeout) {
      clearTimeout(this.saveTimeout)
    }
    
    this.saveTimeout = setTimeout(() => {
      if (this.db) {
        const data = this.db.export()
        const buffer = Buffer.from(data)
        fs.writeFileSync(DB_PATH, buffer)
        console.log('Database saved at', new Date().toISOString())
      }
    }, this.SAVE_DELAY)
  }

  // 强制保存（应用退出时）
  async forceSave() {
    if (this.saveTimeout) {
      clearTimeout(this.saveTimeout)
      this.saveTimeout = null
    }
    if (this.db) {
      const data = this.db.export()
      fs.writeFileSync(DB_PATH, Buffer.from(data))
    }
  }
}
```

export async function runTransaction<T>(
  operation: (db: Database) => Promise<T>
): Promise<T> {
  const db = await getDatabase()
  
  try {
    db.run('BEGIN TRANSACTION')
    const result = await operation(db)
    db.run('COMMIT')
    return result
  } catch (error) {
    db.run('ROLLBACK')
    throw error

}

createTask: async (taskData) => {
  const optimisticId = `temp_${Date.now()}`
  const optimisticTask: Task = {
    ...taskData,
    id: optimisticId,
    createdAt: new Date().toISOString(),
    updatedAt: new Date().toISOString(),
  } as Task

  get().addTask(optimisticTask)

  try {
    const result = await window.electronAPI.createTask(taskData)

      // 替换临时 ID 为真实 ID
      get().updateTask(optimisticId, { 
        id: result.data.id,
        ...result.data

      return result.data
    } else {

      get().deleteTask(optimisticId)
      return null
    }
  } catch (error: any) {
    // 异常回滚

    return null

}

// renderer/src/stores/errorStore.ts
interface ErrorState {
  errors: Array<{
    id: string
    message: string
    type: 'error' | 'warning' | 'info'
    timestamp: string
  }>
  addError: (message: string, type?: 'error' | 'warning' | 'info') => void
  removeError: (id: string) => void
  clearErrors: () => void
}

export const useErrorStore = create<ErrorState>((set) => ({
  errors: [],
  
  addError: (message, type = 'error') => {
    const id = `error_${Date.now()}`
    set((state) => ({
      errors: [...state.errors, { id, message, type, timestamp: new Date().toISOString() }]
    }))
    
    // 3 秒后自动移除
    setTimeout(() => {
      set((state) => ({
        errors: state.errors.filter(e => e.id !== id)
      }))
    }, 3000)
  },
  
  removeError: (id) => set((state) => ({
    errors: state.errors.filter(e => e.id !== id)
  })),
  
  clearErrors: () => set({ errors: [] })
}))
```

getProjectStats: (projectId) => {
  if (statsCache.has(projectId)) {
    return statsCache.get(projectId)
  }

  const tasks = get().tasks.filter(t => t.projectId === projectId)
  const stats = {

// main/src/services/cacheService.ts
class CacheService {
  private cache = new Map<string, any>()

  get<T>(key: string): T | null {
    const entry = this.cache.get(key)
    if (!entry || Date.now() - entry.timestamp > entry.ttl) {
      return null
    }
    return entry.data
  }

  set<T>(key: string, data: T, ttl: number = 5000) {
    this.cache.set(key, { data, timestamp: Date.now(), ttl })
  }
}

export const cacheService = new CacheService()
