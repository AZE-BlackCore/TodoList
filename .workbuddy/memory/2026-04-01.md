## TaskReminder 浮窗拖拽功能

### 问题
任务提醒浮窗固定在右下角，无法移动位置，可能遮挡内容。

### 解决方案
为 `TaskReminder` 组件添加了完整的拖拽功能：

1. **拖拽实现**
   - 使用 `drag-handle` 类标记可拖拽区域（标题栏）
   - 监听 `mousedown`、`mousemove`、`mouseup` 事件
   - 计算鼠标偏移量，实现平滑拖拽
   - 边界检查，防止拖出窗口外

2. **位置持久化**
   - 使用 localStorage 保存位置
   - 下次启动时自动恢复上次的位置
   - 默认位置：右下角

3. **视觉反馈**
   - 拖拽手柄图标（Move 图标）
   - 鼠标变为 `grab` / `grabbing` 状态
   - 拖拽时禁止选中文字（select-none）
   - 按钮点击事件阻止冒泡（stopPropagation）

4. **用户体验优化**
   - 只在标题栏拖拽，不影响内容区
   - 按钮操作不受拖拽影响
   - 流畅的拖拽体验，无延迟

### 文件修改
- `renderer/src/components/notification/TaskReminder.tsx`
  - 添加 `position`、`isDragging`、`dragOffset` 状态
  - 添加 `handleMouseDown`、`handleMouseMove`、`handleMouseUp` 处理器
  - 添加位置加载/保存的 useEffect
  - 容器改为绝对定位，动态控制 left/top
  - 标题栏添加 `drag-handle` 类和拖拽样式

## 数据库导入错误修复

### 问题
创建日程时报错：`(0, database_1.getDb) is not a function`

### 根本原因
`main/src/services/scheduleService.ts` 文件中错误地导入了 `getDb` 函数，但 `database.ts` 中导出的函数名是 `getDatabase`。

### 解决方案
修复了 `scheduleService.ts` 中的所有数据库函数调用：

1. **导入语句修复**
   ```typescript
   // 错误：import { getDb } from './database';
   // 正确：import { getDatabase } from './database';
   ```

2. **函数调用修复**（共 6 处）
   - `createSchedule()` - 第 65 行
   - `getScheduleById()` - 第 101 行
   - `getSchedules()` - 第 134 行
   - `updateSchedule()` - 第 184 行
   - `deleteSchedule()` - 第 256 行
   - `getSchedulesByDate()` - 第 264 行

3. **异步处理**
   - 所有 `getDatabase()` 调用都添加了 `await` 关键字
   - 因为 `getDatabase()` 是异步函数

### 文件修改
- `main/src/services/scheduleService.ts`
  - 修复导入语句
  - 修复所有数据库函数调用
  - 添加异步处理
